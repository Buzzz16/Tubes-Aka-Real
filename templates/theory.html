Teori Kompleksitas Algoritma

Studi Kasus: Bilangan Eksponen (BaseExp)

Pengantar

Dalam ilmu komputer, terdapat berbagai cara untuk menyelesaikan masalah yang sama. Proyek ini membandingkan dua pendekatan untuk menghitung pangkat: Algoritma Iteratif dan Algoritma Rekursif.

1. Algoritma Iteratif

Konsep

Menggunakan perulangan (for loop) untuk mengalikan basis sebanyak n kali.

Kompleksitas Waktu (Time Complexity)

O(n) - Linear. Waktu eksekusi bertambah secara proporsional dengan besarnya pangkat.

Kompleksitas Ruang (Space Complexity)

O(1) - Konstan. Hanya membutuhkan memori untuk menyimpan variabel hasil.

Implementasi Kode

def pangkatIteratif(basis, pangkat):
    result = 1
    for i in range(pangkat):
        result *= basis
    return result

Pseudocode

Function pangkatIteratif(basis, pangkat):
    result ← 1
    For i from 0 to pangkat - 1:
        result ← result × basis
    Return result

2. Algoritma Rekursif

Konsep

Fungsi memanggil dirinya sendiri dengan masalah yang lebih kecil (n-1) hingga mencapai basis (n=0).

Kompleksitas Waktu

O(n) - Linear. Melakukan n kali pemanggilan fungsi.

Kompleksitas Ruang

O(n) - Linear. Membutuhkan stack memory untuk menyimpan setiap pemanggilan fungsi (Call Stack).

Implementasi Kode

def pangkatRekursif(basis, pangkat):
    if pangkat == 0:
        return 1    
    return basis * pangkatRekursif(basis, pangkat - 1)

Pseudocode

Function pangkatRekursif(basis, pangkat):
    If pangkat = 0:
        Return 1
    Return basis × pangkatRekursif(basis, pangkat - 1)

Isu Recursion Overhead

Meskipun kedua algoritma memiliki Time Complexity O(n), secara praktis Iteratif seringkali lebih cepat daripada Rekursif. Hal ini disebabkan oleh Overhead pada Rekursif, yaitu biaya sistem untuk:

  • Membuat stack frame baru untuk setiap pemanggilan fungsi
  • Menyimpan alamat kembali (return address)
  • Menyimpan parameter lokal di memory stack

Untuk membuktikan perbedaan performa ini, silakan lakukan pengujian pada halaman Single Test atau Range & Graph Analysis.
